# Kombucha Tracker

A mobile-first web application for fermentation hobbyists to track SCOBY health and fermentation batches with daily logs, AI-powered photo analysis, AI chat advisor with TTS, onboarding quiz with personalized AI advice, photo management, and fermentation reminders.

## Architecture

### Monorepo Structure
- `artifacts/api-server/` — Express 5 backend API
- `artifacts/kombucha-tracker/` — React + Vite frontend
- `lib/db/` — PostgreSQL database with Drizzle ORM
- `lib/api-spec/` — OpenAPI specification
- `lib/api-zod/` — Zod schemas (generated by Orval)
- `lib/api-client-react/` — React Query hooks (generated by Orval)
- `lib/integrations-openai-ai-server/` — OpenAI client (server-side)
- `lib/object-storage-web/` — Object storage client (web)

### Stack
- **Monorepo**: pnpm workspaces
- **Node.js**: 24
- **TypeScript**: 5.9
- **API**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Auth**: Clerk (via @clerk/express, @clerk/react)
- **AI**: OpenAI via Replit AI Integrations
- **Storage**: Replit Object Storage (GCS)
- **Validation**: Zod v4, drizzle-zod
- **API codegen**: Orval (from OpenAPI spec)
- **Frontend**: React + Vite + Tailwind CSS + shadcn/ui + wouter

## Database Schema

Tables in PostgreSQL:
- `profiles` — user profile (clerkUserId, hasCompletedOnboarding, hasScoby, experienceLevel, flavorPreference, ttsEnabled, etc.)
- `batches` — fermentation batches (userId, name, startedAt, status, teaType, notes)
- `logs` — daily fermentation logs (batchId, dayNumber, temperature, scobylook, smell, color, notes, aiTip)
- `photos` — batch photos (batchId, objectPath, caption, dayNumber, aiAnalysis)
- `chat_messages` — AI chat history (userId, role, content)

## API Routes

All routes are under `/api/`:
- `GET /api/healthz` — health check
- `GET/PUT /api/profile` — user profile
- `GET/POST /api/batches` — list/create batches
- `GET/PUT/DELETE /api/batches/:batchId` — batch CRUD
- `GET/POST /api/batches/:batchId/logs` — batch logs
- `GET /api/batches/:batchId/logs/:logId` — single log
- `GET/POST/DELETE /api/batches/:batchId/photos` — batch photos
- `POST /api/ai/analyze-photo` — AI photo analysis
- `POST /api/ai/onboarding-advice` — personalized onboarding advice
- `POST /api/ai/chat` — AI chat with context
- `GET /api/ai/chat/history` — chat history
- `POST /api/ai/tts` — text-to-speech (returns base64 mp3)
- `GET /api/ai/flavoring-guide` — F2 flavoring suggestions
- `POST /api/storage/uploads/request-url` — presigned upload URL
- `GET /api/storage/objects/*path` — serve private objects
- `GET /api/storage/public-objects/*path` — serve public objects
- `GET /api/dashboard/summary` — dashboard stats

- `/batches/new` — create batch (protected)
- `/batches/:batchId` — batch detail with logs, photos (protected)
- `/batches/:batchId/log` — create daily log (protected)

- `/advisor` — AI chat with TTS (protected)
- `/flavoring` — F2 flavoring guide with preference selector (protected)
- `/settings` — TTS toggle, profile info (protected)

## Key Design Decisions

- **Photo URLs**: `objectPath` stored in DB starts with `/objects/UUID`. The API route `/api/storage/objects/*path` extracts the wildcard and prepends `/objects/` to look up the file. Frontend uses `getPhotoUrl()` helper which strips `/objects/` prefix from stored path.
- **Auth**: requireAuth middleware uses `@clerk/express`'s `getAuth()`. Auth token is set via `setAuthTokenGetter` in App.tsx using `useAuth().getToken`.
- **AI tips**: Generated automatically when creating a log entry.
- **Onboarding**: User redirected to `/onboarding` if `hasCompletedOnboarding=false`. After completing, profile is updated and user goes to `/dashboard`.

## Key Commands

```bash
# Install dependencies
pnpm install

# Push DB schema
pnpm --filter @workspace/db run push

# Start API server
pnpm --filter @workspace/api-server run dev

# Start frontend
pnpm --filter @workspace/kombucha-tracker run dev

# Regenerate API client from OpenAPI spec
pnpm --filter @workspace/api-client-react run codegen

# Type check API server
pnpm --filter @workspace/api-server run typecheck
```

## Environment Variables

- `CLERK_SECRET_KEY` — Clerk server key
- `CLERK_PUBLISHABLE_KEY` — Clerk publishable key (server)
